# ToPu_MeshHistory

**ToPu_MeshHistory** は、Blenderでのメッシュ編集を自動で記録し、制作過程をあとからタイムラプスのように再生できるアドオンです。

頂点の移動だけでなく、トポロジーの変更、UV、カラー属性、頂点グループ・ウェイト、シェイプキー、マテリアル、モディファイアーなど、メッシュに関する変化を履歴として保存します。録画した履歴からタイムラインに連動する「ライブプレビュー」を作成し、再生速度やタイミングを調整できます。

## ダウンロード

### [最新版のダウンロードはこちら](https://github.com/http4211/ToPu_MeshHistory/releases)

リリースページの最新版を開き、`ToPu_MeshHistory_v*.zip` をダウンロードしてください。

## 主な機能

- メッシュ編集をバックグラウンドで自動記録
- アクティブ、選択中、シーン全体から録画対象を選択
- 複数の録画データをプロジェクトごとに管理
- 過去の録画を読み込み、続きを録画
- 録画履歴からタイムライン連動のライブプレビューを作成
- グラフエディターで再生タイミングを調整
- 同じトポロジーが続く区間のメッシュ変化を滑らかに補間
- 録画データを `.blend` の外部へ保存し、ファイルサイズの増加を抑制

## 対応環境

- Blender 4.3 以降

## インストール

1. [Releases](https://github.com/http4211/ToPu_MeshHistory/releases) から最新版のZIPファイルをダウンロードします。
2. ZIPファイルは展開せず、そのまま使用します。
3. ダウンロードしたZIPファイルを、起動中のBlenderの画面へドラッグ＆ドロップします。
4. インストール確認画面が表示されたら、内容を確認してインストールします。
5. 必要に応じて `編集 > プリファレンス > アドオン` を開き、`ToPu_MeshHistory` が有効になっていることを確認します。

ドラッグ＆ドロップを使用しない場合は、`編集 > プリファレンス > アドオン` を開き、右上のメニューから `ディスクからインストール` を選んでZIPファイルを指定することでもインストールできます。

## UIを開く

3Dビューで `N` キーを押し、サイドバーの `ToPu` タブから `ToPu_MeshHistory` を開きます。

トップバーの録画アイコンからも、録画・履歴一覧・ライブプレビューのポップダウンを開けます。

## 基本的な使い方

### 1. Blendファイルを保存する

録画を始める前に、現在の `.blend` ファイルを保存してください。未保存のBlendファイルでは録画を開始できません。

### 2. 録画対象を選ぶ

`録画対象` から、記録するメッシュの範囲を選びます。

| 録画対象 | 内容 |
| --- | --- |
| アクティブメッシュ | アクティブなメッシュを録画します |
| 選択メッシュ | 選択中のメッシュを録画します |
| シーン内の全メッシュ | シーンにあるすべてのメッシュを録画します（デフォルト） |

### 3. 録画を開始する

`新しい録画` を押すと、新しい録画データが作成されます。そのまま通常どおりモデリングを行ってください。メッシュに変化があると自動で履歴へ記録されます。

録画中に確実にチェックポイントを残したい場合は、`現在を記録` を押します。

### 4. 録画を停止する

作業の記録を終えるときは `録画を停止` を押します。停止後、録画データを読み込んだり、続きを録画したり、ライブプレビューを作成したりできます。

### 5. 録画データを読み込む

`録画データ一覧` から使用する録画を選択し、`読み込み` を押します。

一覧では次の操作も行えます。

| 操作 | 内容 |
| --- | --- |
| 録画名をダブルクリック | 表示名を変更します |
| 続きを録画 | 選択した録画の末尾から録画を再開します |
| フォルダーアイコン | 録画データの保存先を開きます |
| ゴミ箱アイコン | 選択した録画データを削除します |
| 更新アイコン | 録画データ一覧を再読み込みします |
| ファイルアイコン | `history.tmh` を直接指定して読み込みます |

### 6. ライブプレビューを作成する

録画データを読み込んだあと、`ライブプレビューを作成` を押します。

録画内のメッシュ状態がGeometry Nodes式のプレビューへ変換され、Blenderのタイムラインに連動して再生できるようになります。作成されたライブプレビューは `.blend` 内に保存されます。

> 録画するには、先に作成済みのライブプレビューを削除してください。

### 7. タイムラインで再生・調整する

ライブプレビューを作成すると、録画の先頭を0%、末尾を100%とする `履歴位置` に開始・終了キーが自動作成されます。タイムラインを再生すると、制作過程をタイムラプスのように確認できます。

| ライブプレビュー設定 | 内容 |
| --- | --- |
| 履歴位置 | 録画のどの時点を表示するかを0〜100%で指定します |
| 現在の録画尺へキーを合わせる | キーの範囲を現在の録画イベント数に合わせ直します |
| メッシュ変化を滑らかに補間 | 同じトポロジーが続く区間の頂点位置を補間します |
| 元オブジェクトを隠す | プレビュー中に元のメッシュを非表示にします |
| 再作成 | 読み込んだ録画データからプレビューを作り直します |
| 削除 | ライブプレビューと履歴位置のタイミングキーを削除します |

`履歴位置` のキーフレームはグラフエディターで自由に編集できます。キーの間隔やカーブを調整することで、再生速度、停止、緩急などを演出できます。

`メッシュ変化を滑らかに補間` は、頂点・辺・面の構成が同じ区間だけに適用されます。トポロジーが変化する区間は、次の状態へ瞬間的に切り替わります。

## 録画の再開

録画データ一覧で対象を選び、`続きを録画` を押すと、既存の履歴の末尾から記録を再開できます。

アドオン設定の `再開時に増分録画` をオンにすると、元の録画データを複製してから続きを記録します。元データを残したまま別テイクとして作業を続けたい場合に便利です。

## 自動開始

`自動開始` はデフォルトでオンです。ただし、新しいBlendファイルで勝手に録画が始まることはありません。

そのBlendファイルで一度 `新しい録画` または `続きを録画` を手動実行すると自動開始が有効になり、次回ファイルを開いたときや保存後に録画を開始します。録画中の状態で保存した場合は、同じ録画データから自動的に再開します。

## アドオン設定

録画欄の歯車アイコンから、Blenderプリファレンスの `ToPu_MeshHistory` 設定を開けます。

| 設定 | 内容 |
| --- | --- |
| 履歴保存先の基準フォルダー | すべてのBlendで共通して使う録画データの保存先です。空欄ではBlendファイルと同じ階層へ保存します |
| 自動開始 | 一度手動録画したBlendを、ファイル読み込み時や保存後に自動録画します |
| 再開時に増分録画 | 元の録画を複製し、複製側へ続きを記録します |
| 記録間隔 | 連続編集中に状態を記録する最短間隔です |
| ディスク同期間隔 | 録画キャッシュをディスクへ確定する間隔です |

## 録画データの保存場所

保存先を変更していない場合、録画データは `.blend` ファイルと同じ階層へ次の構成で保存されます。

```text
ToPu_MeshHistory/
└─ Blendファイル名/
   └─ 録画日時/
      ├─ history.tmh
      ├─ index.jsonl
      └─ session.json
```

- `history.tmh`：メッシュ履歴本体
- `index.jsonl`：履歴を読み込むためのインデックス
- `session.json`：録画名、日時、イベント数などの録画情報

録画を別の環境へ移動するときは、録画日時フォルダー内の3ファイルをまとめて移動してください。

## 注意事項

- 録画できるのはメッシュオブジェクトです。
- 録画データは `.blend` とは別に保存されるため、プロジェクトのバックアップ時は `ToPu_MeshHistory` フォルダーも一緒に保管してください。
- ポリゴン数、録画対象数、履歴イベント数が多いほど、録画データの容量とライブプレビューの作成時間・メモリ使用量が増えます。
- 録画データの削除はBlenderのUndoでは取り消せません。必要なデータを確認してから削除してください。
- 録画データを削除しても、すでに作成したライブプレビューは `.blend` 内に残ります。
